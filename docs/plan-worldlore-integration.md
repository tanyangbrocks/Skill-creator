## 世界觀核心摘要

### 世界架構
- **蒼究**：法則等級世界（凡/超凡/法則/次元/神/絕對六級中第三層），受《位面法》約束
- 世界有意志（蒼天），可修改規則、生成隨機事件
- 宏觀結構：至高神壇 → 無數神界 → 無數世界

### 種族體系
- 四階血脈（黃/玄/地/天），決定成神難度
- 主要體系：具魔種族（人/類人/精怪/寄生/半神）、不具魔種族（萬獸/天行）、次元物種
- 神話種族（三階，需解鎖）
- **人族 vs 鬼族**：伴生制衡對，互以對方為資源，形成三大人族勢力

### 能量體系（十八體系）
```
三大根源：
  修煉六道：武道 / 仙道 / 法道 / 意道 / 魂道 / 詭道（圖騰）
  支配六法：魔法 / 妖力 / 奧術 / 神聖力 / 源能 / 星能
  世界六意：技能 / 職業 / 超能 / 神力 / 概念 / 尋能

12 種 MP 對應 12 種能量（與屬性刻印是不同維度）
同源複合（單一體系極致）vs 異源複合（多體系融合，更均衡強大）
```

### 元素體系
- 7 基礎：金木水火土風雷
- 進階：毒光暗幻冰
- 極元素：超聖沌空等（涉及法則層）
- **碰撞系統**：15+ 種雙元素組合效果（感電/沸騰/燃燒等）+ 三元素合體技

### 法則體系
- 頂層 12 大法則（基軸四 / 天道四 / 根源四）
- 普通生物需藉修煉極境或特殊手段觸及法則之力
- 法則碰撞有優先權鏈

### 境界系統（對應等級）
| 境界 | 等級區間 | 能力點上限 | 說明 |
|------|---------|---------|------|
| 凡人 | LV 1–9 | 50 | 起點 |
| 超凡 | LV 10–29 | 120 | 開始觸及能量 |
| 王者 | LV 30–49 | 250 | 強者之路 |
| 聖人 | LV 50–69 | 500 | 法則感知 |
| 巔峰 | LV 70–99 | 900 | 近神之境 |
| 成神 | LV 100+ | 無上限 | 解鎖神界 |

## 導入計畫（三批）

> 以下為方向性規劃，每一項在實作前需與開發者確認細節。

---

### 第一批：低阻力（直接對應，改動面小）

**W-1｜境界突破節點**
- 現有 XP/等級系統加入 5 個境界閾值（LV10/30/50/70/100）
- 境界突破：給予額外能力點、顯示境界名稱、可選解鎖新圖騰類型
- 改動：`AbilityPointCalculator.ExceedsLevelCap`、`PlayerController`、HUD
- 依賴：無

**W-2｜屬性刻印補完**
- 11 種元素作為 `EngraveElement enum`（金木水火土冰風光暗雷毒）
- 刻印積木加入「屬性選擇」欄位，施放時附加 `DamageTag`
- 目前積木 VM 已有刻印資料結構（`EngraveData`），主要補資料
- 改動：`EngraveData.cs`、`EngraveColor.cs`、`ScratchCanvas.cs`
- 依賴：無

**W-3｜元素碰撞材質表**
- 把世界觀碰撞定義翻譯為 Noita CA 材質互動規則
- 例：水（MaterialType.Water）+ 火（Fire）= 蒸氣（Steam，新材質）
- 改動：`TileWorld.cs` 或獨立 `ElementInteractionTable.cs`
- 依賴：先確認要加哪幾種碰撞效果（建議從最直覺的 5 條開始）

**W-4｜MP 多類型框架（最小版）**
- 擴充單一 `float Mp` → 3 種 MP：靈力 / 魔力 / 真氣
- `PlayerController` 持有三個 MP 槽，各自獨立回復速率
- 法陣設計時指定「消耗哪種 MP」
- 改動：`PlayerController.cs`、`SpellArray.cs`、`SpellCaster.cs`、HUD
- 依賴：W-1（境界決定哪些 MP 槽解鎖）

---

### 第二批：需要設計轉換（需確認機制後才能實作）

**W-5｜法則刻印（高境界限制）**
- 解鎖條件：聖人境（LV50）起
- 優先實作基軸四法則：時間（Wait 倍率）/ 空間（傳送/瞬移）/ 造化（地形生成）/ 因果（觸發鏈）
- 每個法則刻印對應一或多個特殊積木效果
- 設計問題待確認：①法則與屬性刻印可疊加嗎？②法則刻印的能力點代價？

**W-6｜種族選擇**
- 創角時選種族（初期提供 5-8 種，其餘後期解鎖）
- 種族影響：初始屬性偏向、MP 親和度（某種 MP 回復更快或上限更高）
- 設計問題待確認：①種族如何影響積木 VM 的施放效果？②種族是否影響地形互動（如水棲種族在水中移動更快）？

**W-7｜元素碰撞技能觸發**
- 兩個不同屬性的投射物碰撞時，偵測並觸發複合效果
- 例：雷屬投射物 + 水面 = AOE 感電
- 改動：`SpellProjectile.cs` 碰撞偵測 + `ElementInteractionTable.cs`
- 依賴：W-2（屬性刻印）、W-3（材質碰撞）

**W-8｜貨幣 / 採購系統**
- 三種貨幣：靈石（修仙通用）/ 魔晶（魔法陣營）/ 銅錢（凡人日常）
- 敵人掉落、礦物出售、NPC 商人
- 設計問題待確認：匯率是否固定或動態？合成材料用哪種貨幣？

---

### 第三批：長期規劃（大範疇，需獨立模組）

**W-9｜完整 12 MP 體系**
- 從 3 種擴充到 12 種（魂力/精神力/虛能/源能/星能等後期能量）
- 各自修煉路線、儲能瓶（MP 上限擴充道具）機制
- 依賴：W-4 完整版

**W-10｜靈職 / 天賦隨機生成**
- 「靈魂四象 + 體質四象」創角系統
- 固有特性（被動）、能量契合度、靈職（職業天賦能力）
- 體質：血脈、先天體質、素材體、特異體
- 需獨立的角色創建 UI 模組

**W-11｜NPC / 勢力系統**
- 三大人族勢力（永恆聯邦 / 神聖皇朝 / 六龍聖天帝國）
- NPC AI + 派系好感度 + 任務系統
- 依賴：NPC 基礎框架（目前尚無）

**W-12｜神界 / 冥界 / 轉生**
- 成神後（LV100）解鎖神界地圖
- 死亡後進入冥界（可選轉生，保留部分進度）
- 依賴：完整地圖系統 + W-9 + W-10
