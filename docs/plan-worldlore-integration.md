## 蒼究世界觀架構速覽（供實作時查閱）

### 能量體系（MP 來源）
蒼究採用**十八體系三大根源**架構，並支援複合體系（詳見 `戰鬥&三大通用要素&十八體系三大根源.md`、`能量複合理論.md`、`九種修煉同源複合體系.md`、`十五種支配同源複合體系.md`、`二十九種異源複合體系.md`）：

```
基礎十八體系：
  修煉六道：武道 / 仙道 / 法道 / 意道 / 魂道 / 詭道（圖騰）
  支配六法：魔法 / 妖力 / 奧術 / 神聖力 / 源能 / 星能
  世界六意：技能 / 職業 / 超能 / 神力 / 概念 / 尋能

複合體系（18 基礎 + 同源/異源混搭）：
   9 種修煉同源複合（修煉六道兩兩組合）
  15 種支配同源複合（支配六法兩兩組合）
  29 種異源複合（跨根源混搭）

合計：18 + 9 + 15 + 29 = 71 種潛在能量類型（= 71 種 MP）
```

> **架構含義**：MP 類型數量遠超 enum 能維護的規模，必須採資料驅動設計。
> 玩家角色同時可用的 MP 種類有上限（參考 game-setting：一般承受 3 種，特殊可達 5 種，超限自爆）。

### 境界等級
採用**星盟通用戰力等級**（詳見 `星盟通用戰力等級.md`），非 game-setting 文件的通用五境界：

```
正式徽章等級（低→高）：
  學徒級（灰） → 超凡級（黑） → 征戰級（綠） → 主將級（藍）
  → 群星級（黃） → 耀日級（紅） → 巔峰級（紫）

客製化等級（巔峰以上）：
  特異級 → 超特級 → 封王級 → 超王級 → 封帝級 → 超帝級
  → 封聖級 → 超聖級 → 滅世級

→ 遊戲等級（LV 1-100+）與境界的對應數值，留到 W-1 實作時確認
```

### 種族體系
採用**蒼究完整種族設定**（詳見 `種族.md` 及各種族子文件）：
- 四階血脈（黃/玄/地/天），決定成神難度
- 具魔種族（人/類人/精怪/寄生/半神）、不具魔種族（萬獸/天行）、次元物種、神話種族

### 元素與法則
- 元素：7 基礎（金木水火土風雷）+ 進階（毒光暗幻冰）+ 極元素（詳見 `元素.md`）
- 碰撞系統：15+ 種組合效果（詳見 `元素.md`）
- 法則：三千大道頂層 12 大法則（基軸/天道/根源各四，詳見 `法則.md`）

### 長期素材（近期不實作）
- 勢力：三大人族勢力 + 多組織（詳見 `現今勢力.md`）
- 地理：10+ 大陸/域（詳見 `世界觀&地理.md` 及各大陸文件）
- 歷史：人族視角完整時間線（詳見 `歷史(人族視角).md`）
- 文化：裝扮/建築/教育/娛樂/經濟/食物等（詳見各子文件）

### 第二批：核心機制擴充（需要設計決策）

**W-5｜MP 多類型框架**

蒼究共有 **71 種潛在能量類型**（18 基礎 + 9 修煉同源複合 + 15 支配同源複合 + 29 異源複合），
硬編碼 enum 行不通，需要**資料驅動設計**。

架構方向：
- `ManaType`：以 `string id`（如 `"武道"`, `"武仙複合"`）或 `int id`（對應登錄表）取代 enum
- `ManaTypeRegistry`：資料表，每筆記錄包含名稱/根源分類/碰撞規則/親和種族等
- `PlayerController`：持有「已解鎖 MP 槽」（`List<ManaType>` 上限 3 種，特殊 5 種，超限自爆）
- `SpellArray`：指定施放消耗哪種 MP
- 複合體系能量：高境界或特定種族解鎖；初期只登錄部分基礎體系即可運作

實作分階段：
1. **第一階段**（搭配 W-1）：框架建立，先登錄 6 種基礎（每大根源各一代表），確認資料結構可擴充
2. **第二階段**：補完 18 種基礎體系
3. **第三階段**（W-12）：開放複合體系（53 種），依玩家選擇/種族解鎖

實作時讀取：`能量複合理論.md`、`戰鬥&三大通用要素&十八體系三大根源.md`、
            `九種修煉同源複合體系.md`、`十五種支配同源複合體系.md`、`二十九種異源複合體系.md`、
            各能量詳細子文件
改動：新增 `ManaType.cs` / `ManaTypeRegistry.cs`、`PlayerController.cs`、`SpellArray.cs`、`SpellCaster.cs`、HUD
依賴：W-1（境界決定 MP 槽解鎖）、W-7（種族影響親和度）

**W-6｜法則刻印**
- 實作部分基軸法則對應的積木效果（時間/空間/造化/因果為優先）
- 高境界（群星級以上）才能解鎖
- 實作時讀取：`法則.md`，確認每條法則的具體能力描述
- 改動：`BlockType.cs`、`OpCode.cs`、`SpellCompiler.cs`、`ExecutionLoop.cs`
- 依賴：W-1（境界限制）

**W-7｜種族選擇（輕量版）**
- 創角時選種族，影響初始屬性偏向和 MP 親和度
- 第一版只實作主要人族 + 2-3 個常見種族；其餘後期補
- 實作時讀取：`種族.md` + 蒼究世界觀種族子文件
- 改動：新增 `PlayerRace.cs`，`PlayerController.cs` 初始化時注入種族參數

**W-8｜貨幣系統**
- 蒼究貨幣體系（待實作時讀取確認種類）
- 敵人掉落、礦物出售、NPC 商人
- 改動：`Inventory.cs` 或新增 `Wallet.cs`，`EnemyManager.cs`

## 設計原則備忘

1. **蒼究設定 > game-setting 文件**：兩者描述同一概念時，以蒼究版本為準
2. **細節留到實作時讀**：不需要現在把所有設定值背熟，計畫只定義「做什麼、讀哪份文件」
3. **分批獨立可交付**：每一項 W-N 可以單獨完成，不強制連帶
4. **第三批純內容型**：不阻擋遊戲系統開發，隨遊戲成熟度逐步填入
