## 拆分邏輯與遷移腳本

`docs/split-docs.ps1` 是一次性遷移腳本，也是拆分邏輯的可執行規格。
Section mapping 寫在腳本頂部的 `SPLIT LOGIC` 區塊。

若要更改分組規則（例如把某個章節移到不同的子檔），修改那個 mapping 區塊並重新執行即可。

**注意**：遷移完成後，子檔案是 source of truth，直接編輯子檔案，不要重跑腳本（會覆蓋你的修改）。

---

## 歷史歸檔規則

每當 Phase N 完成，將 `實作進度.md` 中對應的 Phase N 詳細段落：
1. 附加到 `docs/history/phase1-2.md`（或新建 `history/phase3.md`）
2. 在 `實作進度.md` 刪除詳細段落，只在「最新完成」表格留一行摘要
